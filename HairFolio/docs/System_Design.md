## Architecture
- **Client-side interface:** Simple input forms or console to record transactions and expenses
- **Backend logic:** Services to store, aggregate, and analyze data
- **Data storage:** Local files or database to persist transactions and expenses
- **Analysis module:** Calculates monthly summaries, trends, and top services

## Data Models
- Transaction: id, date, service, price, notes
- Expense: id, date, type (fixed/short-term), description, amount

## Planned Features
- Monthly financial summary
- Habit-tracker style dashboard for trends
- Exportable reports (CSV/PDF)
- Scalable architecture for future enhancements
